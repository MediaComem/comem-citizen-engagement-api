Usage

Welcome to Citizen Engagement API.

Data Types

The following tables describe the JSON representations of the resources accessible in this API.

Properties marked as read-only are sent by the server in JSON responses but always ignored in requests. Properties marked as write-only can be sent in JSON requests but are never sent back by the server in responses (e.g. password).

PropertyConstraintsDescription

creatorHrefread-onlyA hyperlink reference to the user who reported the issue

tagseach tag: 1-20 charsAn optional array of tag strings describing the issue

get /users

Retrieve a paginated list of users.

The list is sorted by name in alphabetical order by default.

  • Request
  • Response

Headers

  • Authorization: required (string - pattern: ^Bearer .+$)

    Use this header to authenticate with a bearer token.

    Roles required: staff

    You can obtain an authentication token with the /auth resource.

    Example:

    Bearer 09ajewax

Query Parameters

  • page: required (integer - default: 1 - minimum: 1)

    The page you want to retrieve.

    Example:

    2
  • pageSize: required (integer - default: 20 - minimum: 1 - maximum: 50)

    The number of items you want to retrieve per page.

    Example:

    15
  • sort: required (string)

    Sorting criteria for the collection.

    Use a criterion as is for ascending order, e.g. name, or with a minus character for descending order, e.g. -name.

    The following criteria are available for this resource: name, firstname, lastname, phone

    Example:

    sort=lastname&sort=firstname

  • Pagination-Total: required (number)

    The total number of elements available in the collection (if any filters are applied, this is the total number of elements that match the filters).

    Example:

    76

/auth

Authentication resource

/auth post

post

Authenticate with a user's credentials and obtain a bearer token that can be used to authenticate to other API operations.

The generated token will be valid for 15 days. The response will also contain a user property with a representation of the authenticated user.

To authenticate to other resources, send the token in the Authorization header with this format: Authorization: Bearer TOKEN.

/me/issues get

get

Retrieve a paginated list of the issues created by the authenticated user.

The list is sorted by descending creation date (from most recent to oldest) by default.

get /me/issues

Retrieve a paginated list of the issues created by the authenticated user.

The list is sorted by descending creation date (from most recent to oldest) by default.

  • Request
  • Response

Headers

  • Authorization: required (string - pattern: ^Bearer .+$)

    Use this header to authenticate with a bearer token.

    Roles required: any

    You can obtain an authentication token with the /auth resource.

    Example:

    Bearer 09ajewax

Query Parameters

  • page: required (integer - default: 1 - minimum: 1)

    The page you want to retrieve.

    Example:

    2
  • pageSize: required (integer - default: 20 - minimum: 1 - maximum: 50)

    The number of items you want to retrieve per page.

    Example:

    15
  • image: required (string)

    Select only issues with (true) or without (false) images.

    By default, all issues are returned whether they have an image or not.

    Example:

    image=true
  • search: required (string)

    Select only issues matching the specified text.

    The search is case-insensitive and includes the issue's description and tags.

    Example:

    search=foo
  • state: required (string)

    Select only issues in that state.

    Can be used multiple times to find all issues in any of the specified states.

    Example:

    state=new&state=inProgress
  • sort: required (string)

    Sorting criteria for the collection.

    Use a criterion as is for ascending order, e.g. name, or with a minus character for descending order, e.g. -name.

    The following criteria are available for this resource: createdAt, updatedAt, state

    Example:

    sort=state&sort=-updatedAt
  • include: required (string)

    Provide this query parameter one or multiple times to fetch and include linked resources in the response.

    (By default, only hyperlink references are provided in the response, e.g. userHref. Use this parameter to also fetch the corresponding resource in the same API call.)

    The following linked resources can be included with this resource: actions, assignee, creator, issueType

    Example:

    include=creator&include=issueType
